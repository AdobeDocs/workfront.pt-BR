## Níveis de acesso e licenças do Workfront Planning

Você pode atribuir uma licença de Administrador do Planning a um usuário quando adicioná-lo ao Adobe Admin Console como Administrador.

Para obter informações, consulte [Gerenciar usuários na Adobe Admin Console](/help/quicksilver/administration-and-setup/add-users/create-and-manage-users/admin-console.md).

Não é possível modificar o nível de acesso Administrador do Planning no Workfront Planning.

Você pode atribuir os seguintes tipos de licença ao configurar níveis de acesso no Workfront:

* Tipos de licença de fluxo de trabalho
* Tipos de licença do Planning

Os usuários devem ter ambos os tipos de licença atribuídos ao seu nível de acesso para acessar o Workfront Planning.

Você pode atribuir as seguintes licenças do Planning a um Nível de Acesso:

* **Planning Standard**: conceda aos usuários esse acesso se eles precisarem criar e gerenciar todos os objetos do Planning, sem serem administradores.
* **Colaborador do Planning**: conceda aos usuários esse acesso se eles precisarem contribuir com registros, mas não precisarem criar objetos do Planning.

  >[!TIP]
  >
  >O tipo de licença do Colaborador do Planning não estará disponível se sua organização tiver comprado números desiguais de licenças do Workflow e do Planning.


* **Nenhum**: ao atribuir este tipo de licença do Planning, você está removendo especificamente todo o acesso do Planning aos usuários atribuídos a este nível de acesso.

  >[!IMPORTANT]
  >
  >As licenças do Planning e do Fluxo de trabalho funcionam juntas para conceder aos usuários acesso ao Workfront.
  >
  >Você pode conceder a um usuário diferentes níveis de acesso entre o Workflow e o Planning, mas a licença do Workflow não pode ser inferior à licença do Planning.
  >
  >Por exemplo, você pode conceder uma licença do Workflow Standard a um usuário e uma licença do Planning Contributor, mas não pode conceder a ele uma licença do Workflow Contributor e uma licença do Planning Standard.
  >
  >O tipo de licença do Workflow não pode ficar em branco se o usuário precisar acessar o Workflow ou o Planning.

Para obter mais informações, consulte [Visão geral do tipo de licença ao usar o Adobe Workfront Planning](/help/quicksilver/planning/access/license-type-overview.md).

## Configurar nível de acesso

## Atribuir licenças a usuários

Você pode atribuir licenças a usuários ao atribuir a eles um Nível de acesso ao editá-los ou criá-los.

Para obter informações, consulte [Editar perfil de usuário](/help/quicksilver/administration-and-setup/add-users/create-and-manage-users/edit-a-users-profile.md).

## Conceder permissões

Você pode conceder permissões às seguintes entidades no Workfront Planning:

* Espaços de trabalho
* Tipos de registro
* Exibições
* Registros

Para obter mais informações, consulte [Visão geral das permissões de compartilhamento no Adobe Workfront Planning](/help/quicksilver/planning/access/sharing-permissions-overview.md).

O tipo de licença do Adobe Workfront funciona com o tipo de licença do Workfront Planning e com as permissões do Planning para que você tenha acesso a exibir, contribuir ou gerenciar objetos do Workfront Planning.

Para obter informações sobre como os tipos de licença afetam os níveis de permissão para objetos do Workfront Planning, consulte [Visão geral do tipo de licença ao usar o Adobe Workfront Planning](/help/quicksilver/planning/access/license-type-overview.md).

## Compartilhar a área do Planning usando um modelo de layout

<!--First, contact your account manager to obtain access to the current Workfront Planning program.-->

Os usuários padrão e os Administradores do sistema têm as áreas do Planning habilitadas por padrão nas seguintes áreas:

* Menu principal
* Painel esquerdo de projetos, portfólios ou programas

O administrador do sistema deverá adicionar as áreas do Planning a você, se você tiver qualquer outra licença do Workfront e precisar contribuir para o trabalho do Workfront Planning.

O administrador pode adicionar a opção Planning às seguintes áreas modificando e atribuindo você a um modelo de layout:

* Menu principal
* Página de destino
* Painel esquerdo para projetos, portfólios e programas
* Pins

Para adicionar ou remover áreas do Workfront Planning dos usuários da sua instância do Workfront:

1. Faça logon no **Workfront** como administrador do Workfront.

1. Vá para **Menu principal** > **Configuração** > **Interface** > **Modelos de layout** e abra ou crie um modelo de layout.

   Para obter informações sobre como personalizar um modelo de layout, consulte [Criar e gerenciar modelos de layout](/help/quicksilver/administration-and-setup/customize-workfront/use-layout-templates/create-and-manage-layout-templates.md).

1. Atribua o modelo de layout aos usuários que você deseja que tenham acesso ao Workfront Planning.

   Para obter informações, consulte [Atribuir usuários a um modelo de layout](/help/quicksilver/administration-and-setup/customize-workfront/use-layout-templates/assign-users-to-layout-template.md).

   Todos os usuários atribuídos ao modelo agora podem acessar o Workfront Planning no Menu Principal.

   Os usuários podem começar a criar espaços de trabalho, tipos de registros, registros e campos.
