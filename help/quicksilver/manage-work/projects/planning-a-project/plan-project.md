# Planejar uma visão geral do projeto

Um projeto é um grande item de trabalho no Adobe Workfront. Você pode dividir um projeto em tarefas que são itens de trabalho menores, como tarefas. As tarefas podem ser atribuídas a usuários ou equipes que podem ser responsáveis pela conclusão. Quando todos os usuários concluírem suas tarefas, o projeto também se tornará concluído.

Além de planejar suas tarefas em um projeto, muitos outros detalhes vão para o planejamento de projetos. Desde determinar os impactos econômicos até considerar a disponibilidade de recursos, há muitos elementos de projetos que precisam de atenção. 

## Estabelecer um escopo de projeto

O planejamento de projeto significa determinar exatamente quais objetivos principais você precisa realizar para concluir o projeto.

Nas fases iniciais de um projeto, talvez você ainda não tenha determinado se deveria realmente fazer o projeto. Pode não ser rentável ou pode não ter os recursos disponíveis. Nessa etapa de planejamento, é possível criar um projeto no Workfront sem adicionar tarefas e definir o status como planejamento. 

Para obter mais informações sobre como criar um projeto, consulte o artigo [Criar um projeto](../../../manage-work/projects/create-projects/create-project.md).

Considere as seguintes perguntas antes de planejar um projeto: 

* Quais são os objetivos obrigatórios e quais são facultativos?
* Dispõe de tempo suficiente para atingir todos os objetivos ou apenas os objetivos principais?
* Tem orçamento suficiente para realizar todos os objetivos agora? 

Usando o Business Case de um projeto, você pode definir seus objetivos e garantir que seu projeto esteja alinhado à estratégia de sua organização. 

Para obter mais informações sobre como definir um Caso de negócios para um projeto, consulte o artigo [Criar um caso de negócios para um projeto](../../../manage-work/projects/define-a-business-case/create-business-case.md).

Depois de definir o escopo geral do projeto, você pode determinar se deseja realizar o projeto e começar a planejá-lo.

### Definir a data de início e conclusão do projeto {#define-the-start-and-completion-date-of-the-project}

Quando você planeja um projeto, uma das primeiras coisas que precisa determinar é a linha do tempo: quando o projeto pode ser iniciado e quando deve ser concluído. Determine se há um prazo definido para quando o projeto deve ser concluído ou se você deve definir um ponto de partida e trabalhar a partir daí. 

Você pode agendar um projeto a partir de uma Data inicial ou pode agendá-lo a partir da Data de conclusão. Você pode definir esse atributo ao criar o projeto inicialmente e pode alterá-lo a qualquer momento editando o projeto.

### Definir o agendamento do projeto {#define-the-project-schedule}

Você deve determinar um agendamento e associá-lo ao projeto, bem como aos usuários responsáveis pela conclusão das tarefas no projeto. 

Considere definir o seguinte em uma Programação:

* Feriados
* Hora de desligar
* Número de horas disponíveis para trabalho durante um dia ou semana

Saber quando os usuários estão disponíveis para trabalhar é importante para entender o progresso de suas tarefas no projeto. 

Você pode criar várias programações de projeto e aplicar uma programação padrão ao projeto. Agendamentos personalizados adicionais podem ser aplicados a indivíduos diferentes.

Para obter mais informações sobre Agendamentos, consulte o artigo [Criar um agendamento](../../../administration-and-setup/set-up-workfront/configure-timesheets-schedules/create-schedules.md).

### Gerenciar as informações adicionais necessárias para o planejamento do projeto {#manage-additional-information-needed-for-planning-the-project}

Ao planejar um projeto, há informações adicionais que devem ser configuradas antes que você possa começar a trabalhar no projeto. 

Considere fazer as seguintes perguntas:

* Existe um regulamento que obrigue o senhor a manter um registro de eventos e processos? Em caso afirmativo, o que é necessário rastrear?\
   O Workfront permite registrar edições, alterações de escopo, alterações de status e ações para que você possa cumprir os regulamentos específicos do seu setor.\
   Para obter mais informações sobre como definir quais atualizações rastrear no Workfront, consulte o artigo [Atualizações controladas pelo sistema](../../../administration-and-setup/set-up-workfront/system-tracked-update-feeds/system-tracked-update-feeds.md).

* Há alguma informação que você precise rastrear que não tenha um campo no Workfront para armazená-lo?! Em caso positivo, crie Forms personalizado para projetos ou tarefas, onde você pode armazenar essas informações.\
   Para obter mais informações sobre como criar formulários personalizados, consulte o artigo [Criar ou editar um formulário personalizado](../../../administration-and-setup/customize-workfront/create-manage-custom-forms/create-or-edit-a-custom-form.md).

* Existem pontos de verificação de aprovação que devem ser concedidos antes que o trabalho em um projeto possa continuar? Em caso afirmativo, crie Processos de Aprovação para projetos ou tarefas que você pode usar enquanto está criando seu projeto.\
   Para obter mais informações sobre Processos de aprovação, consulte o artigo [Criar um processo de aprovação para itens de trabalho](../../../administration-and-setup/customize-workfront/configure-approval-milestone-processes/create-approval-processes.md).

## Criar a linha do tempo do projeto

Depois de determinar os principais objetivos de um projeto e decidir que o projeto vale a pena perseguir, você deve calcular o cronograma de cada objetivo. Cada objetivo pode se tornar uma de suas tarefas no projeto.

Isso ajuda a priorizar seus objetivos e planejar sua estrutura de detalhamento de trabalho de acordo. A Estrutura de detalhamento de trabalho define a linha do tempo do projeto.\
Para obter mais informações sobre como criar tarefas em um projeto, consulte o artigo [Definir tarefas](#define-tasks).

Considere o seguinte ao criar a linha do tempo do projeto:

* Analise objetivos maiores em tarefas secundárias e defina suas datas de Início e Conclusão.
* Determine se seus objetivos dependem de outros objetivos.

   Você pode configurar essas dependências como antecessores.

   Por exemplo, você pode ter um projeto para construir um prédio de apartamentos. Um de seus objetivos inclui o encanamento e outro é o estabelecimento da base. Uma de suas tarefas de canalização é conectar-se à linha de água da cidade principal, e isso deve ser feito antes da instalação da fundação. No entanto, a maioria de suas outras tarefas de canalização não pode ser feita até que a base seja colocada. Nesse caso, você deve considerar o uso de antecessores para entender quando um objetivo pode começar após a conclusão de seu pré-requisito.

   Ao usar tarefas pai para organizar objetivos, você pode criar uma tarefa para cada objetivo principal e adicionar subtarefas quando chegar ao estágio de dividir objetivos principais em tarefas individuais. Isso mantém as tarefas que fazem parte do objetivo organizado no projeto. 

   Para obter mais informações sobre predecessores de tarefas, consulte [Visão geral dos antecessores de tarefas](../../../manage-work/tasks/use-prdcssrs/predecessors-overview.md).

* Considere os objetivos mais importantes para seu projeto e os sinalize como tarefas de marco.

   Recomendamos usar tarefas pai como marcos.

   Para obter mais informações sobre como usar tarefas de marco, consulte [Criar um caminho de marco](../../../administration-and-setup/customize-workfront/configure-approval-milestone-processes/create-milestone-path.md).

* Defina uma linha do tempo para cada objetivo. Se você trabalhar com tarefas pai, cada tarefa filho deverá ter uma Data de Início e Conclusão definida. O tempo entre a Data inicial mais antiga de uma tarefa e a Data de conclusão mais recente de uma tarefa no mesmo projeto resulta na linha do tempo do projeto. 

## Definir tarefas {#define-tasks}

Ao definir os objetivos do seu projeto e as tarefas associadas a eles, você está criando a linha do tempo do projeto. 

Você pode criar tarefas em um projeto das seguintes maneiras:

* Adicione tarefas a um projeto em uma edição em linha.
* Use um modelo com tarefas de modelo definidas e adicione-o ao seu projeto. 

   As Tarefas do modelo se tornam as Tarefas do projeto. 

   Para obter mais informações sobre como criar tarefas, consulte o artigo [Criar tarefas em um projeto](../../../manage-work/tasks/create-tasks/create-tasks-in-project.md).

Ao definir tarefas, considere o seguinte:

* Defina a linha do tempo de cada tarefa. Isso é capturado no campo Duration de cada tarefa.

   Para obter informações sobre a Duração da tarefa, consulte o artigo [Visão Geral da Duração da Tarefa e do Tipo de Duração](../../../manage-work/tasks/taskdurtn/task-duration-and-duration-type.md).

* Defina a relação pai-filho entre as tarefas.
* Defina a relação do antecessor entre as tarefas.
* Associe um Caminho de marcos ao projeto e associe um Marco à sua respectiva tarefa. 

   Para obter mais informações sobre o uso de tarefas de marco, consulte o artigo [Criar um caminho de marco](../../../administration-and-setup/customize-workfront/configure-approval-milestone-processes/create-milestone-path.md).

* Defina a quantidade de trabalho necessária para concluir cada tarefa. Isso é capturado no campo Horas Planejadas de cada tarefa.

   Para obter mais informações sobre Horas Planejadas, consulte o artigo [Visão geral das Horas Planejadas](../../../manage-work/tasks/task-information/planned-hours.md).

* Atribua cada tarefa a um usuário ou equipe responsável por concluí-la.
* Verifique a disponibilidade dos usuários que você está atribuindo às tarefas. Certifique-se de que eles sejam livres para trabalhar e não sobrealocados para que possam concluir suas tarefas atribuídas. Se os usuários estiverem com alocação excessiva ou tiverem tempo de folga em suas programações, considere um dos seguintes:

   * Reduza as Horas Planejadas de cada tarefa.
   * Adicione mais usuários a uma tarefa para garantir que ela possa ser concluída durante o tempo alocado para ela.
   * Atribua novamente as tarefas a usuários que estão disponíveis sem outras restrições.\
      Para obter mais informações sobre o planejamento dos recursos do projeto, consulte o artigo [Planejamento de recursos no Adobe Workfront](../../../resource-mgmt/resource-planning/resource-planning-overview.md).\
      Para obter mais informações sobre como programar seus recursos para realizar o trabalho em um projeto, consulte [Visão Geral do Balanceador de Carga de Trabalho](../../../resource-mgmt/workload-balancer/overview-workload-balancer.md).
