# Referência de qualidade de escrita

Regras detalhadas para a habilidade `writing-quality`. O arquivo SKILL.md abrange as regras mais comuns; esse arquivo fornece a referência completa para pontuação, terminologia, exemplos de uso de maiúsculas e minúsculas e formatação de procedimento.

&#x200B;---

## Pontuação

### Períodos

- Use pontos em uma lista se os itens concluírem uma frase (&quot;No menu X, você pode:&quot;)
- Usar pontos em uma lista se qualquer item for uma frase completa
- Os pontos não devem ser formatados com o conteúdo principal (Correto: Clique em **Concluído**. / Incorreto: Clique **Concluído.**)
- Usar períodos em extensões de arquivo (.pdf)

### Vírgulas

- Vírgula de Oxford sempre: &quot;motivo, meios e oportunidade&quot;
- Vírgula após frase introdutória: &quot;Para abrir um documento, clique em Abrir&quot;.
- Vírgula após o ano em uma data no meio da frase: &quot;O 2 de janeiro de 2016, edição de...&quot;
- Vírgula antes de &quot;then&quot; nos procedimentos: &quot;Clique em **Configuração** e em **Interface**.&quot;
- Sem vírgula com mês/ano apenas: &quot;Janeiro de 2016&quot;

### Apóstrofos

- Uso em contrações: não, eles são, é
- Singular possessivo: add &#39;s (computador de Jen)
- Terminação possessiva plural em s: adicionar apenas &#39; (O computador dos Jones)
- Diferencie &quot;it&#39;s&quot; (it is) de &quot;its&quot; (belonging to it)

### Dois pontos

- Letras minúsculas depois de dois pontos, a menos que sejam seguidas por aspas diretas, substantivo adequado ou lista com marcadores
- Usar no final de uma frase/fragmento introduzindo uma lista
- Nenhum sinal de dois pontos após os cabeçalhos do procedimento
- Não há dois pontos para introduzir elementos visuais ou segmentos de código

### Ponto e vírgula

- Conectar duas cláusulas sem uma conjunção
- Frases separadas contendo sua própria pontuação
- Não colocar um ponto e vírgula em negrito após o texto em negrito nos procedimentos
- Não incluir em um hiperlink

### Aspas

- Referencie-os como &quot;aspas&quot; (não &quot;aspas&quot;)
- As aspas de fechamento saem de vírgulas e pontos
- A colocação do ponto de interrogação depende se a citação ou a sentença é a questão

### Hífens

- Evite estruturas que exijam hifens quando possível
- Hifenizar modificadores compostos antes de um substantivo: &quot;memória somente leitura&quot;
- Hifenizar quando um modificador for um particípio: &quot;disco protegido contra cópia&quot;
- Hifenizar quando um modificador for um número: &quot;fonte de 12 pontos&quot;
- Nenhum hífen nas combinações de teclas — use + (Ctrl+I)

### Traços eme

Use com moderação. Considere reestruturar a sentença em vez disso.

### Traços

- Intervalos: 2005-2013
- Sinal de menos ou números negativos
- Conecte um adjetivo composto a outra palavra: &quot;opções do tipo caixa de diálogo&quot;

### Reticências

- Código ou sintaxe omitidos
- Indicação da interface do usuário de que uma caixa de diálogo será exibida
- Texto omitido entre aspas (adicione um ponto se substituir o fim da frase)

### Barra

- Não há barra para escolhas — use &quot;ou&quot; (&quot;ele&quot; e não &quot;ele/ela&quot;)
- Uso para combinações: interruptor liga/desliga
- Usar para endereços de Internet e frações

## Exemplos de uso de maiúsculas

### Termos do Workfront — Referência direta versus indireta

| Termo | Direto (capitalizar) | Indireto (minúsculas) |
|------|---------------------|----------------------|
| Data de conclusão planejada | &quot;Preencha o campo **Data de conclusão planejada**.&quot; | &quot;Cada tarefa deve ter uma data de conclusão planejada.&quot; |
| Modelo de Layout | &quot;Na Instalação, vá para **Modelos de Layout**.&quot; | &quot;Atribuir um modelo de layout a cada usuário.&quot; |
| Administrador de Grupo | &quot;Localize o campo **Administrador de Grupo**.&quot; | &quot;Entre em contato com o administrador de grupo.&quot; |
| Formulário personalizado | &quot;Anexe o **Formulário personalizado** ao projeto.&quot; | &quot;Você pode criar um formulário personalizado para qualquer objeto.&quot; |

### Termos comuns do Workfront (use letras maiúsculas ao fazer referência à interface do usuário)

- Formulário personalizado
- Duração
- Horas planejadas
- Data de conclusão planejada
- Tipo de hora
- Administrador de grupo/Administrador do sistema
- Administrador do Workfront (a pessoa, sempre &quot;administrador&quot; em letras minúsculas)
- Notificações de Lembrete
- Modelo de Layout
- Filtro, Visualização, Agrupamento (no contexto de Filtros, Visualizações e Agrupamentos)
- Planejador de recursos (planos)

### Botões

- Seguir capitalização da interface
- Se o botão for all-caps na interface do usuário, escreva-o em maiúsculas e minúsculas

## Terminologia

### Elementos da interface

| Elemento | Termo correto |
|---------|-------------|
| Lista de opções flexíveis | Menu (adicione &quot;menu suspenso&quot; para maior clareza quando necessário) |
| Comandos indisponíveis | &quot;indisponível&quot; (nunca &quot;esmaecido&quot;, &quot;desativado&quot; ou &quot;acinzentado&quot;) |
| Pop-up com campos de formulário | Caixa de diálogo |
| Botões sem nome | Usar o nome da dica de ferramenta do botão |
| Mensagens de erro / notificações em caixa de informações | &quot;Mensagem&quot; (&quot;Uma mensagem é exibida notificando você...&quot;) |
| Entrada de texto | Campo |

### Funções

| Contexto | Termo |
|---------|------|
| A função no Workfront | Administrador de sistema |
| A pessoa que administra o Workfront | administrador do Workfront |

### Substituir vs. Substituir

Estas são palavras diferentes, com significados diferentes - use a correta para o contexto.

### Extensões de arquivo

Sempre use a extensão de arquivo, não a sigla: &quot;Carregar um arquivo .pdf&quot; (não &quot;Carregar um PDF&quot;).

## Formatação do Procedimento

### Apresentações

Inclua somente quando necessário. Formatos preferidos (em ordem):

1. Frase infinita + dois pontos: &quot;Para criar uma senha temporária para um usuário em seu sistema:&quot;
2. Conclusão da frase + ponto: &quot;Você deve criar senhas temporárias para os usuários do seu sistema&quot;.
3. Frase infinita + &quot;siga estas etapas&quot; + dois pontos: &quot;Para criar uma senha temporária para um usuário, siga estas etapas:&quot;

### Convenções de etapa

- Use listas numeradas para todos os procedimentos, mesmo os de etapa única
- Ações clicáveis em negrito e elementos de interface do usuário nas etapas
- Usar verbos de ação específicos: &quot;digitar&quot;, &quot;selecionar&quot;, &quot;clicar&quot; — evitar &quot;definir&quot; ou &quot;especificar&quot;
- Descreva as respostas do sistema antes que ocorram ou imediatamente depois

Exemplo de posicionamento de resposta do sistema:

```
1. Click **Sharing**.
2. In the **Task Access** dialog box, enter the name of the user you want to give task access to.
```

### Etapas condicionais e opcionais

```
3. (Conditional) If you are a member of more than one team, select...
3. (Optional) To add an emoji, click...
3. (Conditional and optional) If you want to add a note...
```

### Subetapas e Listas de Campos

Introduza com um comando imperativo (Configurar, Especificar, Fazer), depois liste os campos:

```
1. Specify the following:
   a. **Favorite color**: Select your favorite color from the drop-down menu.
   b. **Favorite song**: Type your favorite song.
```

### Vários métodos

Documente a maneira mais simples primeiro:

```
1. In the **Main Menu**, click **Home**.

   Or

   Click the **Home** icon in the upper-left corner.
```

### Colchetes angulares direitos

- Mostrar navegação de menu/guia: &quot;Clique em **Email** > **Notificações**&quot;
- Nomes de botão em negrito, não colchetes
- Espaços ao redor de colchetes
- Não usar para menu principal — use o trecho do menu principal

## Referência em Negrito e Itálico

### Negrito — Usar para:

| Contexto | Em etapas | Etapas Externas |
|---------|----------|---------------|
| Botões/links clicáveis | Negrito | Não negrito |
| Nomes de menu | Negrito | Não negrito |
| Nomes de ícones | Negrito | Não negrito |
| Nomes da caixa de diálogo | Negrito | Não negrito |
| Nomes de página | Negrito | Não negrito |
| Nomes de campos em listas de campos | Negrito + dois pontos | Não negrito |

### Itálico — Utilização para:

- O texto que o usuário deve digitar: &quot;Digite *my.workfront.com* na caixa de URL&quot;
- Nunca por ênfase — reestruturar a frase em vez disso

## Notas, Dicas, Avisos — Orientação detalhada

### Quando usar cada

**Nota** (`>[!NOTE]`):
- Informações que não se encaixam no contexto de parágrafo
- Evite frustração do usuário ou perda de tempo (&quot;Você deve ter uma licença de equipe para concluir esta tarefa&quot;).
- Informações de controle de versão ou compatibilidade (&quot;Esta funcionalidade está disponível somente no Visualizador de provas herdado.&quot;)

**Dica** (`>[!TIP]`):
- Sugestões que facilitam a vida do usuário, mas não são cruciais
- Exemplo: &quot;Você pode compartilhar vários documentos ao mesmo tempo mantendo a tecla Shift pressionada.&quot;

**Importante** (`>[!IMPORTANT]`):
- Informações vitais para a etapa ou procedimento

**Aviso** (`>[!WARNING]`):
- Alertar o usuário sobre possível perda de dados

### Regras

- Máximo de um por seção
- Nunca empilhar várias notas/dicas/avisos
- Combinar notas relacionadas em uma com uma lista com marcadores
- Não usar anotações para anunciar novos recursos — retrabalhe o texto do artigo

## Padrões de referência cruzada

| Situação | Padrão |
|-----------|---------|
| Depois de dar as informações | &quot;Para obter mais informações, consulte [Nome do artigo] (link).&quot; |
| Sem informação prévia | &quot;Para obter informações, consulte [Nome do artigo] (link).&quot; |
| Para uma seção em outro artigo | &quot;Para obter mais informações, consulte [Seção] em [Nome do artigo] (link).&quot; |
| Para uma seção no mesmo artigo | &quot;Para obter mais informações, consulte a [Seção] neste artigo.&quot; |

Posicionamento:
- Em parágrafos: em linha
- Em etapas do procedimento: nova linha após a etapa
- Várias referências: lista com marcadores

## Características de qualidade do DQTI

### Fácil de usar

| Característica | Perguntas principais |
|---------------|---------------|
| Orientação da tarefa | Isso se concentra no que o usuário pode fazer? O cabeçalho é um verbo imperativo? |
| Precisão | A terminologia corresponde exatamente à interface do usuário? Os detalhes técnicos estão corretos? |
| Integralidade | Há informações suficientes para agir? Há informações desnecessárias a serem removidas? |

### Fácil de entender

| Característica | Perguntas principais |
|---------------|---------------|
| Clareza | Cada frase é uma ideia? As palavras são simples e diretas? |
| Concretização | Há exemplos específicos? Os elementos da interface do usuário são nomeados exatamente? |
| Estilo | Voz ativa? O tempo presente? Terminologia consistente? |

### Fácil de encontrar

| Característica | Perguntas principais |
|---------------|---------------|
| Organização | Estrutura lógica? Divulgação progressiva? Mais recente primeiro, onde aplicável? |
| Capacidade de recuperação | Um leitor pode digitalizar cabeçalhos e entender o conteúdo? Os links são fornecidos? |
| Eficácia visual | Listas para mais de 3 itens? Tabelas para comparações? Não há parágrafos na parede de texto? |

## Correções comuns

| Problema | Correção |
|-------|-----|
| &quot;Estamos entusiasmados em apresentar...&quot; | Remover idioma de marketing — declare o recurso diretamente |
| &quot;A configuração do X foi atualizada&quot; | &quot;Agora você pode configurar X&quot; (ativo, orientado a tarefas) |
| &quot;Recomenda-se que...&quot; | &quot;Recomendamos...&quot; |
| &quot;Os usuários podem utilizar o recurso para...&quot; | &quot;Você pode usar o recurso para...&quot; |
| &quot;definir o valor&quot; / &quot;especificar a opção&quot; | &quot;digite o valor&quot; / &quot;selecione a opção&quot; |
| &quot;desabilitado&quot; / &quot;esmaecido&quot; | &quot;indisponível&quot; |
| &quot;dropdown&quot; (independente) | &quot;menu&quot; ou &quot;menu suspenso&quot; |
| &quot;ele/ela&quot; | &quot;eles&quot; ou reestruturar |
| Notas empilhadas | Combinar em uma nota com marcadores |
| Etapas de procedimento externo em negrito | Remover negrito |
