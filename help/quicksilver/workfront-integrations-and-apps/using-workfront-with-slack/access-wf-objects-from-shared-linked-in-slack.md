# Acessar [!DNL Adobe Workfront] objetos de um link compartilhado em [!DNL Slack]

Quando links para objetos do [!DNL Adobe Workfront] são compartilhados com você em um canal do [!DNL Slack], você pode exibir informações adicionais sobre eles no [!DNL Slack], após instalar e configurar o [!DNL Workfront for Slack].

Para obter mais informações sobre a configuração de [!DNL Workfront for Slack], consulte [Configurar [!DNL Adobe Workfront for Slack]](../../workfront-integrations-and-apps/using-workfront-with-slack/configure-workfront-for-slack.md).

Você deve estar conectado ao [!DNL Workfront] em uma guia de navegador separada para abrir o link ou executar ações adicionais nesses objetos a partir de [!DNL Slack].

Quando um usuário está compartilhando um link para um objeto no [!DNL Workfront] com você, informações adicionais sobre o objeto também podem ser exibidas junto com o link. O usuário que está enviando o link deve ter pelo menos [!UICONTROL permissões de Exibição] para que as informações adicionais sejam exibidas em [!DNL Slack].

## Pré-requisitos

Antes de poder acessar seus favoritos e itens recentes do [!DNL Slack], você deve

* Configurar [!DNL Workfront for Slack]\
   Para obter instruções sobre como configurar o [!DNL Workfront for Slack], consulte [Configurar [!DNL Adobe Workfront for Slack]](../../workfront-integrations-and-apps/using-workfront-with-slack/configure-workfront-for-slack.md).

## Acessar projetos de [!DNL Slack]

1. Faça logon na instância do [!DNL Slack] e no [!DNL Workfront from Slack.]

   Para obter mais informações sobre como fazer logon no [!DNL Workfront] a partir de [!DNL Slack], consulte a seção &quot;Fazendo Logon no [!DNL Workfront from Slack]&quot; no [Access [!DNL Adobe Workfront from Slack]](../../workfront-integrations-and-apps/using-workfront-with-slack/access-workfront-from-slack.md).

1. Navegue até uma mensagem que inclua um link para um projeto [!DNL Workfront].

   >[!NOTE]
   >
   >O usuário que está compartilhando o link deve ter pelo menos [!UICONTROL permissões de Exibição] para o projeto para que você veja informações adicionais sobre ele.

1. (Opcional) Clique no nome do projeto para abri-lo em [!DNL Workfront] em uma nova guia do navegador.
1. Considere revisar as seguintes informações adicionais sobre o projeto:

   * **[!UICONTROL Status]:** Para obter mais informações sobre o status do projeto, consulte [Acessar a lista de status de projeto do sistema](../../administration-and-setup/customize-workfront/creating-custom-status-and-priority-labels/project-statuses.md).

   * **[!UICONTROL Data de conclusão planejada]**: para obter mais informações sobre a Data de conclusão planejada, consulte [Definir a Data de conclusão planejada do projeto](../../manage-work/projects/planning-a-project/project-planned-completion-date.md).

   * Nome da **[!UICONTROL Portfolio]**: para obter mais informações sobre portfólios, consulte [Entender a metodologia de portfólio](../../manage-work/portfolios/portfolios-overview/portfolio-overview.md).

   * **[!UICONTROL Condição]**: para obter mais informações sobre a Condição do Projeto, consulte [Visão Geral da Condição do Projeto e do Tipo de Condição](../../manage-work/projects/manage-projects/project-condition-and-condition-type.md).

   * **[!UICONTROL Percentual concluído]**: para obter mais informações sobre o valor do Percentual concluído de um projeto, consulte [Visão geral do Percentual concluído do projeto](../../manage-work/tasks/task-information/project-percent-complete.md).

   * **[!UICONTROL Número de Referência]**: Para obter mais informações sobre números de referência de objeto, consulte [Usar o número de referência de objetos](../../workfront-basics/navigate-workfront/search/reference-number-of-objects.md).

1. (Opcional) Clique em **[!UICONTROL Assinar]** para assinar o projeto.

   Você receberá uma confirmação de que se inscreveu com êxito no projeto.

## Acessar tarefas de [!DNL Slack]

1. Faça logon na instância do [!DNL Slack] e no [!DNL Workfront from Slack].

   Para obter mais informações sobre como fazer logon no [!DNL Workfront] a partir de [!DNL Slack], consulte a seção &quot;Logon no [!DNL Workfront] a partir de [!DNL Slack]&quot; no [Access [!DNL Adobe Workfront from Slack]](../../workfront-integrations-and-apps/using-workfront-with-slack/access-workfront-from-slack.md).

1. Navegue até uma mensagem que inclua um link para uma tarefa [!DNL Workfront].

   >[!NOTE]
   >
   >O usuário que está compartilhando o link deve ter pelo menos [!UICONTROL permissões de Exibição] para a tarefa para que você veja informações adicionais sobre ela.

1. (Opcional) Clique no nome da tarefa para abri-la no [!DNL Workfront] em uma nova guia do navegador.
1. Considere a revisão das seguintes informações adicionais sobre a tarefa:

   * **[!UICONTROL Status]**

     Para obter mais informações sobre status de tarefas, consulte [Acessar a lista de status de tarefas do sistema](../../administration-and-setup/customize-workfront/creating-custom-status-and-priority-labels/task-statuses.md).

   * **[!UICONTROL Percentual concluído]**
   * **[!UICONTROL Atribuído A Nome]**
   * **[!DNL Planned Completion Date]**
   * **[!UICONTROL Nome do Projeto]** ou **[!UICONTROL Nome da Tarefa Pai]**
   * **[!UICONTROL Número de Referência]**

     Para obter mais informações sobre números de referência de objeto, consulte [Usar o número de referência de objetos](../../workfront-basics/navigate-workfront/search/reference-number-of-objects.md).

1. (Opcional) Clique em **[!UICONTROL Assinar]** para assinar a tarefa.

   Você receberá uma confirmação de que se inscreveu com êxito na tarefa.

1. (Opcional) Clique em **[!UICONTROL Atribuir ao usuário]** para atribuir a tarefa a um usuário.
1. (Condicional) Comece digitando o nome de um usuário, se você clicou em **[!UICONTROL Atribuir ao usuário]**.

   Você não pode atribuir uma tarefa a uma função de trabalho ou equipe de [!DNL Slack].

## Problemas de acesso do [!DNL Slack]

1. Faça logon em sua instância do [!DNL Slack] e em [!DNL Workfront] a partir de [!DNL Slack].

   Para obter mais informações sobre como fazer logon no Workfront a partir de [!DNL Slack], consulte a seção &quot;Fazendo Logon no [!DNL Workfront] a partir de [!DNL Slack]&quot; no [Acesso [!DNL Adobe Workfront] a partir de [!DNL Slack]](../../workfront-integrations-and-apps/using-workfront-with-slack/access-workfront-from-slack.md).

1. Navegue até uma mensagem que inclua um link para um problema [!DNL Workfront].

   >[!NOTE]
   >
   >O usuário que está compartilhando o link deve ter pelo menos [!UICONTROL Exibir] permissões para o problema para que você veja informações adicionais sobre ele.

1. (Opcional) Clique no nome do problema para abri-lo no [!DNL Workfront] em uma nova guia do navegador.
1. Considere a revisão das seguintes informações adicionais sobre a tarefa:

   * **[!UICONTROL Status]**: para obter mais informações sobre status de problemas, consulte [Acessar a lista de status de problemas do sistema](../../administration-and-setup/customize-workfront/creating-custom-status-and-priority-labels/issue-statuses.md).

   * **[!UICONTROL Atribuído A Nome]**
   * **[!UICONTROL Data de Término Planejada]**
   * **[!UICONTROL Nome do projeto]**
   * **[!UICONTROL Número de Referência]**: Para obter mais informações sobre números de referência de objeto, consulte [Usar o número de referência de objetos](../../workfront-basics/navigate-workfront/search/reference-number-of-objects.md).

1. (Opcional) Clique em **[!UICONTROL Assinar]** para assinar o problema.

   Você receberá uma confirmação de que se inscreveu com êxito no problema.

1. (Opcional) Clique em **[!UICONTROL Atribuir ao usuário]** para atribuir o problema a um usuário.
1. (Condicional) Comece digitando o nome de um usuário, se você clicou em **[!UICONTROL Atribuir ao usuário]**.

   Você não pode atribuir um problema a uma função de trabalho ou equipe de [!DNL Slack].
